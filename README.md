# gostdoc

CLI и Python-библиотека: приводят оформление `.docx` (доклад, курсовая, ВКР, отчёт о НИР)
к **ГОСТ 7.32-2017**. Меняется только оформление — текст автора остаётся побайтно прежним.

## Установка (для разработки)

```
python -m venv .venv
.venv\Scripts\activate
pip install -e .[dev]
```

## Использование

```
gostdoc INPUT.docx [-o OUTPUT.docx] [--check]
```

- Без `-o` результат пишется рядом: `INPUT.gost.docx` (исходник не перезаписывается).
- `--check` ничего не пишет: проверяет соответствие ГОСТ-константам, печатает несоответствия,
  код выхода `0` (соответствует) или `1` (есть отклонения).

## Что делает v0

Поля, шрифт, кегль, межстрочный интервал, абзацный отступ, выравнивание, нумерация страниц,
оформление существующих стилей заголовков и подписей — по ГОСТ 7.32-2017.

Чего v0 **не** делает: не распознаёт неразмеченные заголовки, не правит список литературы,
не учитывает методички вузов, не конвертирует `.doc` → `.docx`, не трогает орфографию.

См. `ТЗ.md` и `CLAUDE.md`.
